hexo+github简要搭建流程

发表于 2016-12-30 | 分类于 实用教程 |

本教材简单的教一下如何用hexo+github搭建博客，过程有点简单，中间可能会出现各种各样的问题，我把自己遇到的问题以及解决方法都有写出来，当然只是我自己遇到过的，对于文中不明白或理解的地方大家可以自行去谷歌百度，网上有非常多的教程。

部署hexo到github上

Git,node的安装

这一部分大家可自行去网上查找，很简单。

安装hexo

  • 右键打开安装好的Git Base，是git的。
  • 安装hexo：npm install -g hexo
  • 安装hexo相应的文件：hexo init （在你想放置文件的路径下执行此命令，也可以cd到那）
  • 安装依赖包： npm install
  • 启动本地预览：
    1. hexo g：用于生成静态文件
    2. hexo s：用于启动服务器，主要用来本地预览
    3. 访问浏览器http://localhost:4000就可以看到本地预览了

创建github仓库

首先注册一个github帐号，然后新建一个仓库，注意一点：仓库名要和你的github名字一样，不然会失败。我就是名字不一样，卡了好久。

部署本地hexo到github仓库

在博客安装位置找到_config.yml文件，用记事本打开，在文件最下方添加：
deploy:
type: git
repository: https://github.com/tengj/tengj.github.io.git
branch: master

注意每个冒号:后面都要加一个空格

配置ssh

  1. ls -al ~/.ssh：检查电脑有没有安装
  2. ssh-keygen -t ras -C "1292104744@qq.com"：ssh绑定邮箱，邮箱是你注册github的邮箱
  3. ssh-agent -s：输入命令进行下一步
  4. ssh-add ~/.ssh/id rsa：输入命令进行下一步
  5. 如果上面两个命令出现错误，请输入这两个指令：eval `ssh-agent -s`；ssh-add
  6. clip<~/.ssh/id_ras.pu：复制ssh钥匙的命令
  7. 打开github→settings→SSH keys：在key下面粘贴刚才的ssh钥匙

    上面的命令不懂得具体意思可自行百度

最后的启动

  • 先安装hexo-deployer-git这个模块：npm install hexo-deployer-git --save
  • 启动本地预览：hexo g；hexo s
  • 启动github端预览：hexo g；hexo d
  • 接着会有帐号密码框出现，输入github帐号密码
  • 打开你的网址名也就是仓库名字：xxxx.github.io即可成功连接

主题更换

  • cd到博客根目录下，这里我选择安装的是next主题（因为有详细的官方教程，用的人也很多）
  • 安装主题： git clone https://github.com/iissnan/hexo-theme-next themes/next
  • 修改hexo根目录下的_config.yml文件 theme: next
  • cd themes/next （cd到主题目录）
  • git pull ：更新配置
  • hexo g；hexo d
  • 具体大家可以到next主题官网查看，有详细的教程。

hexo常用的命令

  • hexo init [folder] 新建一个网站，没设置folder，默认当前文件夹
  • hexo g 用于生成静态文件
  • hexo s 用于启动服务器，主要用来本地预览
  • hexo d 用于将本地文件发布到github上
  • hexo new “文章名” 用于新建一篇文章
  • hexo new page “页面名” 用来创建页面
  • npm update -g hexo 更新hexo
  • cd到主题文件夹：git pull 更新主题
  • npm update 更新插件
  • hexo clean 清除缓存文件和已生成的静态文件

常见问题

  • 部署到github是首页中文乱码：把站点配置文件_comfig.yml另存为编码为utf-8的文件覆盖之前即可
  • hexo s命令执行完之后无法打命令：按ctrl+c即可
  • 本地预览没问题，部署到github页面空白时可清楚缓存试下hexo clean然后清楚浏览器缓存，然后等。。。
  • 新建的文章不显示：查看文档，看看标签、分类后面有没有加空格，曾经没注意到这个问题搞了很久
  • 文件名字有’%’会出现错误
  • 标签分类冒号后面都要加空格
  • hexo d错误：deploy的type改成git，然后运行下npm install hexo-deployer-git –save 再hexo g hexo d

总结

本教程很多地方都不够详细，有很多点都没说到，以后有时间在做补充，尤其是不知道自己命令是否成功了没，博主没有把命令正确时的图给贴出来，等下次重新安装时再把详细图给贴出来，大家不懂的尽量百度和谷歌，基本上都能解决问题的。
本教程对于初入小白来说可能有点看不懂，还是那句话，大家不懂的尽量百度谷歌，也是考验一下自己吧。
