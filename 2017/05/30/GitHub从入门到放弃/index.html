GitHub从入门到放弃

发表于 2017-05-30 | 分类于 实用教程 |

本篇文章简单的简介了git的常用语法，如何Pull Request对方仓库，如何把本地代码push到Github上，如何使用GitHub Pages部署自己的个人项目和个人博客，教程没有图片展示，只有文字描述，大家遇到问题尽量去谷歌百度，不要怕遇到英文网站，看不懂还有谷歌翻译。

Github入门

GitHub 是为开发者提供 Git 仓库的托管服务。这是一个让开发者与
朋友、同事、同学及陌生人共享代码的完美场所。

基本操作

  • git init–初始化仓库
  • git status–查看仓库的状态
  • touch README.md–创建文件
  • git add xxx.md–向暂存区中添加文件
  • git add .–表示添加新文件和编辑过的文件
  • git add -A .–表示添加所有内容
  • git commit -m “文档说明”–保存仓库的历史记录，文档说明必须要写，不然无法提交
  • git log–查看提交日志

分支的操作

  • git branch–显示所在分支
  • git checkout -b fa–创建fa分支并切换到它
  • git checkout master–切换到master分支
  • git merge –no-ff fa–合并fa分支
  • git log –graph–以图表形式查看分支

更改提交的操作

  • git reset –hard 嘻哈值–回溯到嘻哈值的历史
  • git relfog–查看当前仓库执行过的操作日志，可查看嘻哈值
  • git commit -am “概述”–这个命令等价于git add和git commit两个操作

推送至远程仓库

  • git remote add origin git@github.com:你的github名字/项目名字（例如：xiaoxiongbushuo/first-pr）.git–git会自动将这个远程仓库的名称设置为origin(标识符)
  • git push -u origin master–当前分支本地仓库内容推送给远程仓库master
  • git push -u origin fa–将本地分支fa推送到远程仓库，远程仓库多了一个fa

从远程仓库获取

  • git clone git@github.com:你的github名字/项目名字（例如：xiaoxiongbushuo/first-pr）.git –获取远程仓库，要换到其他目录，不要与之前操作的仓库同一目录
  • git checkout -b fa origin/fa–将fa分支获取到本地仓库

    -b 参数的后面是本地仓库中新建分支的名称。让它与远程仓库的对应分支保持同名。新建
    分支名称后面是获取来源的分支名称。

  • git push–把现在所在的本地仓库提交到同名远程仓库
  • git pull origin fa–把远程仓库fa的最新状态更新到本地仓库

Pull Request

Pull Request是自己修改源代码后，请求对方仓库采纳该修改时采取的一种行为。

使用Pull Request流程

  • 在github上进行Fork
  • 将1（上面一行）的仓库clone至本地开发环境
  • 在本地环境中创建特性分支
  • 对特性分支进行代码修改并提交
  • 将特性分支Push到1的仓库中
  • 在GitHub上对Fork来源仓库发送Pull Request
  1. Fork：点击Fork按钮创建自己的仓库
  2. clone:用git复制项目到本地：git clone git@github.com:你的github名字/项目名字（例如：xiaoxiongbushuo/first-pr）.git
  3. 确认分支：git branch -agit要cd到分支文件夹
  4. 创建特性分支：在github上发送Pull Request时，一般都是发送特性分支，创建一个名为work的分支，用来发送Pull Request，这个work分支就是这次的特性分支git checkout -b work gh-pages
  5. 确认是否切换到work分支下：git branch -a
  6. 查看文件列表：ls
  7. 添加代码：打开要修改的文件进行修改
  8. 提交修改：git diff查看修改是否已经正确进行
  9. 将修改的文件添加至本地仓库：git add 修改文件名.后缀 ;git commit -m "Add my impression"
  10. 创建远程分支：git push origin work(也可以说是把本地代码推到远程work分支)
  11. 把远程最新代码更新到本地master：git pull origin master
  12. 查看分支：git branch -a,
  13. 登录GitHub并切换到work分支，点击绿色按钮，会跳转到查看分支间差别的页面。确认内容没差别后，点击Create Pull Request提交
  14. 之后可以登录GitHub，打开控制面板查看Pull Request。
  15. 明确标出“正在开发过程”：在Create Pull Request时，可以在标题前面加[WIP]字样，表示仍在开发过程中。

本方法适用于从github上面clone下来的项目，不适合本地上传到github

GitHub的SSH绑定本地的SSH

一般下载git的时候会自带SSH，没有的话自行下载，可在git输入SSH查看有没有安装

本地生成SSH

首先在Git Badh输入ssh-keygen -t rsa，就是指定 rsa 算法生成密钥，接着连续三个回车键（不需要输入密码），然后就会生成两个文件 id_ras 和 id_rsa.pub ，而 id_rsa 是密钥，id_rsa.pub 就是公钥。这两文件默认分别在如下目录里生成：

Linux/Mac 系统 在 ~/.ssh 下，win系统在 /c/Documents and Settings/username/.ssh 下，都是隐藏文件，相信你们有办法查看的。

接下来要做的是把 id_rsa.pub 的内容添加到 GitHub 上，这样你本地的 id_rsa 密钥跟 GitHub 上的 id_rsa.pub 公钥进行配对，授权成功才可以提交代码。

GitHub上添加SSH key

  1. 在GitHub上的设置页面，点击左侧的SSH and GPG keys，然后电机右上角的New SSH key按钮
  2. 在Key那一栏把 id_rsa.pub 公钥文件里的内容复制粘贴进去，最后电机Add SSH key按钮
  3. 输入ssh -T git@github.com 进行测试添加SSH key是否成功

本地项目Push到github仓库

假设我们本地有一个test2项目，我们需要在github上建一个test项目，然后把本地test2上的所有代码commit记录提交到github上的test项目上

  1. 在GitHub上建一个test项目
  2. 把本地test2项目与github上的test项目进行关联，切换到test2目录
    git remote add origin git@github.com:你的github名字/test.git--git会自动将这个远程仓库的名称设置为origin(标识符)
  3. 查看当前项目有哪些：git remote -v
  4. 提交远程代码：git push -u origin master

进行上一步时会有错误提示，这是因为把github上的README.md文件没有在本地，我们执行git pull –rebase origin master命令把.md文件拷贝到本地就可以解决错误了

进行push是提示Everything up-to-date时，是因为我们没有git add和git commit，解决方式为输入正确的add和commit:(git add xxx.md–向暂存区中添加文件 git add .–表示添加新文件和编辑过的文件 git add -A .–表示添加所有内容)3个选一个使用，然后git commit -m “提交信息概述”

用GitHub Pages展示个人项目

用腾讯云或百度云部署自己的作品太麻烦，我们可以使用github pages展示自己的网页作品，非常简单实惠，我们默认已经创建好了一个项目xiaoxiong

  1. 首先准备好自己作品的源代码
  2. 打开项目中的Settings，在GitHub Pages那一栏的Source选项中选择把none换成master branch，然后点击旁边的Save，刷新页面时GitHub Pages那一栏会显示一个网址，那就是我们展示页面的网址
  3. 打开电脑，创建一个文件夹，空白处右键git bash here，把刚才创建的仓库clone下来，命令我们上面有说到：git clone git@github.com:你的用户名/你的项目名.git，然后CD到所clone的文件夹目录
  4. 接下来把自己要展示的代码文件放置到这个文件夹中
  5. 我们可以用git status查看一下仓库中文件是否为红色成功添加进去，然后git add .把所有文件添加到暂存区，再次输入git status查看所有文件变绿色代表添加成功
  6. 输入git commit -m "概述"把暂存区的文件实际保存到仓库的历史记录中
  7. 输入git push把所有文件push到github远程仓库，如果是第一次提交需要输入github帐号密码
  8. 最后打开之前GitHub Pages显示的网址，一般为：https://你的用户名.github.io/你的项目名/或者https://你的用户名.github.io.你的项目名/你根目录中的html文件名字（例如index.html）
